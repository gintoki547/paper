Project Logic

不是复古装饰，而是把古籍变成可读的城市系统

项目以“时间推进”和“主题分层”为总框架：前五张图串联汴京一日生活，第六张图补全年节民俗，让观众从具体场景进入宋代城市文化。

01

原典依据

以《东京梦华录》原文为主体，结合注释本和学术研究，保留地点、时辰、人物、器物、活动等可回溯信息。

02

结构化整理

将笔记体叙述拆解为空间、时辰、饮食、服饰、节庆等字段，避免只凭画面氛围进行想象式设计。

03

视觉转译

用地图、罗盘、分类图表、图标和场景插画建立阅读路径，让观众知道先看什么、再理解什么。

04

媒介延展

从横版信息图继续延展到纸品、龙鳞装、集章卡、钥匙扣、动态视频和网站游历，形成完整传播方案。

文本收集

阅读原文、注释和研究资料，摘录地点名称、时间表达、人物身份、器物名称、饮食种类、服饰形制和节庆场景。

Data Evidence

数据不是装饰，是作品的证据链

仓库中的数据表对应六张信息图的不同主题，页面用摘要方式呈现字段逻辑和统计重点。

图一早市最终数据集 / 图5_州桥夜食_食物清单.csv

饮食消费数据

记录食物名称、类别、关联店铺、地理位置、营业时段、烹饪方式、温度属性和味觉分类，用于支撑晨食与夜食两张图。

字段方法

  • 原文摘录保留出处，保证信息可回溯。
  • 地点、时辰、身份、器物和行为拆成字段。
  • 统计结果只服务图面阅读，不虚构精确历史数字。
  • RAWGraphs、Flourish 与手绘图面共同完成视觉转译。

Visual System

统一的标题字、矿物色、地图和罗盘

视觉系统保留传统文化气质，但服务信息阅读，避免停留在古风装饰。

字体系统

标题字以手写书法形成识别，正文与注释强调清晰度，建立主题气质和阅读效率之间的平衡。

图标与图表

桥梁、城门、酒楼、瓦舍、蒸笼、幞头等元素被转化为统一线性图标，辅助地图、分类和说明。

地图与罗盘

地图负责空间导览，罗盘负责时辰和节令提示，使六张图在横版长卷中保持连续阅读关系。

赭石 石青 石绿 朱砂 蛤白 墨色

Extensions

从展板到可翻阅、可收藏、可互动的文创系统

延展设计不是简单套图，而是将六张信息图中的节令、饮食、地图、标题字和人物场景转化为不同媒介。

纸品延展

将节令、人物和市井场景转化为月份纸品，保留日期功能并加入节气、花卉、饮食和人物活动。
